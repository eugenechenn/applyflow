# Resume Structuring Examples

## Good Case 1：工作经历

输入：
```text
2025.11-2026.02 广东公诚管理公司 项目管理助理
负责项目进度跟踪、资料整理和风险预警
协助推进流程标准化建设
```

输出：
```json
{
  "company": "广东公诚管理公司",
  "role": "项目管理助理",
  "timeRange": "2025.11-2026.02",
  "bullets": [
    "负责项目进度跟踪、资料整理和风险预警",
    "协助推进流程标准化建设"
  ]
}
```

## Good Case 2：项目经历

输入：
```text
2025.03-2025.10 华南理工大学 企业诊断小组组长
带领小组完成企业经营诊断
输出问题梳理与改进建议
```

输出：
```json
{
  "name": "华南理工大学 企业诊断小组",
  "role": "组长",
  "timeRange": "2025.03-2025.10",
  "bullets": [
    "带领小组完成企业经营诊断",
    "输出问题梳理与改进建议"
  ]
}
```

## Good Case 3：教育背景

输入：
```text
2024.09-2026.06 华南理工大学 工商管理专业
2016.09-2020.06 广东技术师范大学 通信工程专业
```

输出：
```json
[
  {
    "school": "华南理工大学",
    "major": "工商管理专业",
    "timeRange": "2024.09-2026.06"
  },
  {
    "school": "广东技术师范大学",
    "major": "通信工程专业",
    "timeRange": "2016.09-2020.06"
  }
]
```

## Bad Case 1：把每句都当工作经历

错误输入：
```text
负责项目进度跟踪
协助推进标准化建设
整理资料并做复盘
```

错误输出：
```json
[
  { "company": "负责项目进度跟踪" },
  { "company": "协助推进标准化建设" }
]
```

正确做法：
- 这些句子应当作为某一条工作经历的 bullets
- 如果没有 header，不要强行造公司名

## Bad Case 2：项目经历误进技能区

错误输入：
```text
企业诊断项目，负责问题梳理与改进方案输出
```

错误输出：
```json
{
  "skills": ["企业诊断项目，负责问题梳理与改进方案输出"]
}
```

正确做法：
- 这是项目或项目 bullet，不是技能

## Bad Case 3：自我评价混入工作经历

输入：
```text
逻辑思维清晰，学习能力强，能够快速适应新技术
```

正确输出：
```json
{
  "selfEvaluation": [
    "逻辑思维清晰，学习能力强，能够快速适应新技术"
  ]
}
```

禁止输出：
- `workExperience`
- `projectExperience`

## Empty Case：没有项目经历

如果原始简历没有项目经历：
- `projectExperience` 可以是空数组
- 不允许为了页面完整性编造项目
