# Change Boundary Checklist

每一轮修改前必须先确认：

1. 本轮唯一目标是什么  
   只能有一个主目标，不能同时做功能、重构、中文化、部署修复。

2. 本轮只允许改哪些文件  
   建议不超过 5 个核心文件。

3. 本轮属于哪个主层级  
- parser
- structuring/schema
- workspace/view-model
- render/UI

4. 哪些旧路径必须删除  
   不能只新增新逻辑而不删旧逻辑。

5. 哪些文件明确不动  
   避免改动扩散。

6. 回滚方式是什么  
   至少说明：
- 回滚到哪个 commit / branch
- 或撤销哪一组文件

7. 验收门是什么  
   必须是可观察结果，不是“感觉更合理”。
